Sitespeed.io 2.1 release notes

This release focus on one thing: Break builds for Jenkins & Travis-CI if your browser timings doesn't match your limits.

JUnit support for Browser timings

Added an easy way of include sitespeed.io in Travis-CI

Minor changes

  • Prepared for HTTP2.0 rules & renamed the current rulesets, new names: sitespeed.io-desktop & sitespeed.io-mobile
  • Output the the input parameters to the error.log fle so it is easy to reproduce the error
  • Centralized the error logging
  • Made it possible to analyze sites with non signed certificates
  • Finetuned the logo
  • Bug fix: The crawler sometimes picked up URL:s linking to other content types than HTML
  • Bug fix: The JUnit XSLT outputted timings metrics

See the changelog for changes done in the past.
