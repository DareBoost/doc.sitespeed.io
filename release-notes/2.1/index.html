Sitespeed.io 2.1 release notes

This release focus on one thing: Break builds for a href="http://jenkins-ci.org/" target="_blank">Jenkins and Travis-CI if your browser timings doesn't meet your limits.

Break builds if your pages are too slow

Configure the limits for your pages and break builds if the timings are higher than your configuration. You can use the timings in the Navigation Timing API or create your own custom timings. Also choose how many times you want to test the page and choose if you want to choose the median or different percentile values.

An easy way of include sitespeed.io in your project & test using Travis-CI

Drop the latest sitespeed.io release in your project, configure the Travis config file and break the build if it doesn't meet your limits. Checkout the example project on Github. Now you can break builds if you break your web performance best practices rules or if you exceed your timing limits. Running with Travis, you can test your site timings using Firefox!

Minor changes

  • Prepared for HTTP 2.0 rules & renamed the current rulesets, new names: sitespeed.io-desktop & sitespeed.io-mobile
  • Output the the input parameters to the error.log file so it is easy to reproduce the error
  • Centralized the error logging
  • Made it possible to analyze sites with non signed certificates
  • Finetuned the logo
  • Bug fix: The crawler sometimes picked up URL:s linking to other content types than HTML
  • Bug fix: The JUnit XSLT outputted timings metrics

See the changelog for changes done in the past.
