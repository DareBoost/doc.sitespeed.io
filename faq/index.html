FAQ

Q: Are there any known bugs?

Checkout the https://github.com/soulgalore/sitespeed.io/issues?labels=bug&state=open list for latest updated list of defects within sitespeed.

There are a couple of inherited bugs from PhantomJs/Yslow:

  • Favicon don't fetch the right response cache headers (so right now in sitespeed, the favicon rule is disabled)
  • The gzipped size of a component is not working so right now the unzipped size is showed everywhere. See original defect here. However, the document size in sitespeed is displayed correctly.

If you found other bugs, please report them here.

Q: I want to add my own rules, how do I do that?

Check out the documentation.

Q: Is there a option to exclude a given directory?

Yes, kind of, checkout http://sitespeed.io/documentation/#run. Use the option -s that will skip urls that contains the specified text in the path.

Q: I want to test only one page, how do I do that?

$ ./sitespeed.io -u http://mysite.com -d 0

Q: It takes like forever to to analyze my site, what should I do?

You can change the amount of simultaneously processes that analyze your site (default is five). Increase the amount of processes with the p switch:

$ ./sitespeed.io -u http://mysite.com -p 10

Q: Can I run sitespeed on Windows?

You can use Git Bash.
