WPO tools - How to get your site faster

So you got a bad score & your site loads slow, what to do then? Here's a collection of tools to speed up your site:

  • Tools to analyze your web page
  • Images
  • CSS images
  • Combine javascript and css files
  • Mobile
  • Asynchronously load javascript
  • Minimize backend time
  • Other nice tools

Analyze your website

Before you start changes things, you need to get metrics, so you can compare what you actually do. There are a lot of tools out there that you can use to analyze a page performance. The best one is http://www.webpagetest.org/, created by Patrick Meenan and supported by Google. With this tool you get everything you need; waterfall charts form all over the world, for different devices. It is amazing that a tool like this exist and that it is free.

The other two great tools (apart from sitspeed.io ;)) are YSlow (originally developed by Yahoo, created by Steve Souders and now maintained by Marcel Duran) and Google:s Page Speed Insights.

Other free nice tools to use is http://www.showslow.com/, http://gtmetrix.com/ and http://tools.pingdom.com/fpt/ (wow, a Swedish company that cares about speed!).

Images

One thing that you really want to avoid, is to deliver things to the end user that isn't necessary, and you don't want to deliver images that are larger in size than necessary. My favorite tool is ImageOptim (Mac only), that really can decrease the size and still have good quality. If you don't use a Mac, you can always use http://smush.it, a great online tool to decrease the size of your images. Also remember to use progressive Jpegs, read this great article by Ann Robson.

CSS images

Everybody knows it but not everyone do it, create a CSS Sprite that contains all of your CSS images in one file (check out this sprite that Google uses in it Play store).

I like two online tools that helps you creates your sprites: http://spriteme.org/ and http://www.spritecow.com/

Combine javascript and css files into as few files as possible

One things is for sure, you don't want to combine the files by hand, you want a tool that let you keep the files separated when developing and then combine for your production environment.

Wordpress users should use W3 Total Cache (that is almost the only thing you need to speed up your performance, fantastic plugin for Wordpress) or WP Minify.

If you are using Java, the maven plugin minify-maven-plugin is good, you need to add a couple of other plugins to make it convenient to use, you can check out an example here.

If you are using Jekyll, you can do it either by yourself creating one include file each (one for CSS and one for JS) or you can use a plugin: Jekyll asset bundler and compress them by https://github.com/stereobooster/jekyll-press.

Mobile

Before you start doing anything for mobile, I think reading this article Make your mobile pages render in under one second by Bryan McQuade is the best start. Focus on what is really important, the speed :)

Testing things on mobile devices is a little harder, Steve Souders has compiled the best bookmarklet for testing mobile performance.

Not performance related but I really like http://quirktools.com/screenfly, a tool where you can easy change the view port, to see how your site looks at different devices.

Another cool tool is Throttle that helps you test how your webapp performs on poor network connections.

Asynchronously load javascript

The number one most important thing with speed & javascript is to load it asynchronously. You can do that yourself or use a javascript loader (and make sure the loader also is loaded asynchrounsly). Here is a spreadsheet of javascript loaders and an example of a good javascript loader is ControlJS. If you really want to optimize (and you want to do that) then load your js async and non onload blocking!

Minimize backend time

One really important thing is to minimize the backend time on your server. You do that by caching! Cache cache cache. There are 1000 of different ways of doing that, depending on your technology stack, but before you do that, test the application to see what backend time you have. One way to do it is to check your Time To First Byte. You do that with sitespeed.io, webpagetest.org or use a specific tool like http://www.bytecheck.com/.

Other nice tools

If you need a to check the HTTP headers online, use http://redbot.org/. If you need statistics for how the most popular sites are doing, check out http://httparchive.org/. Also checkout this great list of web performance tools on Steve Souders blog.
